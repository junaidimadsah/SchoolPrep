## 🧭 How to use this repository

1. **Fork or clone** this repository to your own GitHub account.
2. **Track your progress** by checking off the boxes (`[x]`) as you complete each day.
3. **Organize your code** by keeping your daily exercises in folders (e.g., `/week-1/day-1`).
4. **Publish your progress** by enabling GitHub Pages in your repository settings (the theme is already configured at the top of this file!).

## 🗓️ The Roadmap

<details>
<summary><strong>📅 Week 1: The Visual Foundation (HTML & CSS)</strong></summary>

> **Goal:** Understand how web pages are structured and styled.  
> **Tools:** Notepad++ / Sublime Text (Days 1-2), VS Code (Days 3-5)

* [ ] **Day 1: The Raw Structure**
  * **Resource:** [MDN Web Docs - HTML Basics](https://developer.mozilla.org/en-US/docs/Learn/Getting_started_with_the_web/HTML_basics)
  * **Task:** Open Notepad++ or Sublime Text. Write a basic HTML5 boilerplate.
  * **Practice:** Create an article about a favorite topic using `<h1>` to `<h6>`, `<p>`, `<strong>`, and `<em>`.
* [ ] **Day 2: Media and Links**
  * **Resource:** [W3Schools - HTML Links & Images](https://www.w3schools.com/html/html_links.asp)
  * **Task:** Create a navigation menu using `<ul>` and `<li>`.
  * **Practice:** Build an index page that links to 3 other HTML pages. Embed an image and a YouTube video.
* [ ] **Day 3: Transition to VS Code & Intro to CSS**
  * **Resource:** [VS Code Docs](https://code.visualstudio.com/docs/introvideos/basics) & [MDN - CSS First Steps](https://developer.mozilla.org/en-US/docs/Learn/CSS/First_steps)
  * **Task:** Install VS Code and the "Live Server" extension.
  * **Practice:** Write inline, internal, and external CSS. Change text colors, background colors, and font families.
* [ ] **Day 4: The CSS Box Model**
  * **Resource:** [CSS-Tricks - Box Model](https://css-tricks.com/the-css-box-model-introduction/)
  * **Task:** Understand the difference between margin (outside) and padding (inside).
  * **Practice:** Create a grid of "Character Cards" (like in a gacha game) with proper spacing and borders.
* [ ] **Day 5: Week 1 Mini-Project**
  * **Resource:** [freeCodeCamp - Build a Website Tutorial](https://www.freecodecamp.org/news/html-css-tutorial-build-a-website/)
  * **Task:** Build a static Wiki page for a game like *Honkai: Star Rail*.
  * **Requirements:** Include a navigation bar, character images, formatted lore text, and clean CSS styling.

</details>

<details>
<summary><strong>📅 Week 2: Programming Logic & Algorithms (Python)</strong></summary>

> **Goal:** Learn how a computer "thinks" using variables, loops, and conditions.  
> **Tools:** Anaconda (Jupyter Notebook / Spyder)

* [ ] **Day 6: Python Setup & Basics**
  * **Resource:** [Anaconda Docs](https://docs.anaconda.com/anaconda/getting-started/) & [Real Python - Variables](https://realpython.com/python-variables/)
  * **Task:** Understand variables, strings, integers, and floats.
  * **Practice:** Write a script that prints a welcome message and calculates basic math operations.
* [ ] **Day 7: User Input & Conditionals**
  * **Resource:** [W3Schools - Python If...Else](https://www.w3schools.com/python/python_conditions.asp)
  * **Task:** Ask the user for their age and output different messages based on the result.
  * **Practice:** Build a "Power Level Calculator" that outputs a character's tier.
* [ ] **Day 8: Loops (Doing things repeatedly)**
  * **Resource:** [Real Python - "while" Loops](https://realpython.com/python-while-loop/) & ["for" Loops](https://realpython.com/python-for-loop/)
  * **Task:** Write a loop that counts from 1 to 100.
  * **Practice:** Create a simulated "gacha pull" loop running 10 times to drop "3-star", "4-star", or "5-star" items.
* [ ] **Day 9: Data Structures (Lists & Dictionaries)**
  * **Resource:** [Python Official Tutorial - Data Structures](https://docs.python.org/3/tutorial/datastructures.html)
  * **Task:** Store multiple items in a single variable.
  * **Practice:** Create a dictionary of 5 characters and their damage multipliers.
* [ ] **Day 10: Week 2 Mini-Project**
  * **Resource:** [GeeksforGeeks - Console Game Python](https://www.geeksforgeeks.org/text-based-adventure-game-in-python/)
  * **Task:** Build a Terminal-Based RPG Battle Simulator.
  * **Requirements:** Use `while` loops for the battle, `if` statements for HP checks, and `input` to choose "Attack" or "Heal".

</details>

<details>
<summary><strong>📅 Week 3: Databases & The Classic Backend (XAMPP & PHP)</strong></summary>

> **Goal:** Learn the standard SMK Telkom web stack (RDBMS and Server-Side Scripting).  
> **Tools:** XAMPP (Apache & MySQL), VS Code

* [ ] **Day 11: Introduction to XAMPP & SQL**
  * **Resource:** [XAMPP Tutorials](https://www.apachefriends.org/faq_windows.html) & [w3schools - SQL Intro](https://www.w3schools.com/sql/sql_intro.asp)
  * **Task:** Start Apache/MySQL in XAMPP. Open phpMyAdmin.
  * **Practice:** Create a database named `inventory_db` and a table named `items`.
* [ ] **Day 12: SQL Queries**
  * **Resource:** [SQL Bolt - Interactive Lessons 1-5](https://sqlbolt.com/)
  * **Task:** Use the SQL tab in phpMyAdmin.
  * **Practice:** Insert 5 records into your `items` table. Query items costing > 1000 gold.
* [ ] **Day 13: Intro to PHP**
  * **Resource:** [PHP.net - Simple Tutorial](https://www.php.net/manual/en/tutorial.php)
  * **Task:** Create `index.php` inside `C:\xampp\htdocs`.
  * **Practice:** Print dynamic HTML based on PHP variables (e.g., date and time).
* [ ] **Day 14: Connecting PHP to MySQL**
  * **Resource:** [W3Schools - PHP MySQL Connect](https://www.w3schools.com/php/php_mysql_select.asp)
  * **Task:** Connect your `index.php` to `inventory_db`.
  * **Practice:** Fetch data from `items` and display it in an HTML `<table>` using a `while` loop.
* [ ] **Day 15: Week 3 Mini-Project**
  * **Resource:** [Traversy Media - PHP CRUD Guide](https://www.freecodecamp.org/news/php-crud-tutorial/)
  * **Task:** Build a functional CRUD Web App.
  * **Requirements:** Create an HTML form to add a new item, and a button to delete an item.

</details>

<details>
<summary><strong>📅 Week 4: Modern JavaScript & APIs (NodeJS & Postman)</strong></summary>

> **Goal:** Separate the frontend from the backend by building RESTful APIs.  
> **Tools:** Node.js, VS Code, Postman

* [ ] **Day 16: NodeJS Setup & NPM**
  * **Resource:** [Node.js Official Intro](https://nodejs.org/en/learn/getting-started/introduction-to-nodejs)
  * **Task:** Install Node.js. Initialize a project using `npm init -y`.
* [ ] **Day 17: Introduction to Express.js**
  * **Resource:** [Express.js - Hello World](https://expressjs.com/en/starter/hello-world.html)
  * **Task:** Install Express (`npm install express`).
  * **Practice:** Create a simple server on port 3000 responding with "Hello World".
* [ ] **Day 18: Building JSON Endpoints**
  * **Resource:** [MDN Web Docs - JSON](https://developer.mozilla.org/en-US/docs/Learn/JavaScript/Objects/JSON)
  * **Task:** Configure Express to send a JSON array instead of HTML.
* [ ] **Day 19: Testing APIs with Postman**
  * **Resource:** [Postman - Sending Requests](https://learning.postman.com/docs/sending-requests/requests/)
  * **Task:** Install Postman.
  * **Practice:** Create a `POST` route in Node.js. Use Postman to send JSON data and log it in the terminal.
* [ ] **Day 20: Week 4 Mini-Project**
  * **Resource:** [LogRocket - REST API Node/Express](https://blog.logrocket.com/build-basic-api-node-js-express/)
  * **Task:** Build a "Player Stats" REST API.
  * **Requirements:** Build `GET` and `POST` endpoints for players. Test via Postman.

</details>

<details>
<summary><strong>📅 Week 5: Enterprise Architecture (.NET)</strong></summary>

> **Goal:** Prepare for high-concurrency, professional-grade system development.  
> **Tools:** Visual Studio Community, .NET SDK

* [ ] **Day 21: Visual Studio & C# Basics**
  * **Resource:** [Microsoft Learn - C# Beginners](https://learn.microsoft.com/en-us/shows/csharp-fundamentals-for-beginners/)
  * **Task:** Install Visual Studio Community.
  * **Practice:** Create a Console App script using C# Classes and Objects.
* [ ] **Day 22: .NET Core Web API**
  * **Resource:** [Microsoft Learn - Web API ASP.NET Core](https://learn.microsoft.com/en-us/training/modules/build-web-api-aspnet-core/)
  * **Task:** Create a new ASP.NET Core Web API project.
  * **Practice:** Explore the default `WeatherForecast` controller.
* [ ] **Day 23: Controllers and Routing**
  * **Resource:** [Microsoft Learn - Routing ASP.NET](https://learn.microsoft.com/en-us/aspnet/core/mvc/controllers/routing)
  * **Task:** Add a new Controller (`TransactionController`).
  * **Practice:** Create endpoints returning hardcoded models (Account ID & Balance).
* [ ] **Day 24: High-Performance Concepts**
  * **Resource:** [Microsoft Learn - async/await](https://learn.microsoft.com/en-us/dotnet/csharp/asynchronous-programming/)
  * **Task:** Update controller methods to be asynchronous.
  * **Practice:** Add `Task.Delay` to understand non-blocking requests.
* [ ] **Day 25: Week 5 Mini-Project**
  * **Resource:** [Code Maze - ASP.NET Core API](https://code-maze.com/net-core-series/)
  * **Task:** Build a Bank Transaction API prototype.
  * **Requirements:** Endpoints to "Check Balance" and "Deposit Funds". Test via Postman.

</details>

<details>
<summary><strong>📅 Week 6: Version Control, Java, & Deployment</strong></summary>

> **Goal:** Learn how professionals manage code, collaborate, and push to the live internet.  
> **Tools:** Git, Netbeans (Java), FileZilla, GitHub Pages

* [ ] **Day 26: Introduction to Git**
  * **Resource:** [Pro Git Book (Ch 1 & 2)](https://git-scm.com/book/en/v2)
  * **Task:** Install Git. Run `git init`.
  * **Practice:** Use `git add .` and `git commit -m "Initial commit"`.
* [ ] **Day 27: GitHub & Remote Repositories**
  * **Resource:** [GitHub Docs - Adding Local Code](https://docs.github.com/en/get-started/getting-started-with-git/adding-locally-hosted-code-to-github-using-git)
  * **Task:** Create a GitHub account & empty repository.
  * **Practice:** Link local Git to GitHub (`git remote add origin`) and push (`git push -u origin main`).
* [ ] **Day 28: Java Basics (Netbeans)**
  * **Resource:** [W3Schools - Java Tutorial](https://www.w3schools.com/java/)
  * **Task:** Install Netbeans & Java JDK. Create a Java project.
  * **Practice:** Write a simple Java program utilizing Classes.
* [ ] **Day 29: Web Hosting & FileZilla**
  * **Resource:** [FileZilla Official Docs](https://wiki.filezilla-project.org/Using)
  * **Task:** Sign up for a free web host.
  * **Practice:** Upload your Week 3 HTML/CSS/PHP files via FTP to the live internet.
* [ ] **Day 30: Portfolio Setup**
  * **Resource:** [GitHub Guide - Mastering Markdown](https://guides.github.com/features/mastering-markdown/)
  * **Task:** Create a `README.md` showcasing what you've learned.
  * **Practice:** Pin your best projects to your GitHub homepage.
* [ ] **Day 31: Bonus - Jekyll Themes in Action!**
  * **Resource:** [GitHub Docs - Adding a Theme](https://docs.github.com/en/pages/setting-up-a-github-pages-site-with-jekyll/adding-a-theme-to-your-github-pages-site-using-jekyll)
  * **Task:** Watch the YAML Front Matter at the top of this file go to work.
  * **Practice:** Ensure GitHub Pages is enabled in your repository settings. Once deployed, GitHub will read the `theme: jekyll-theme-hacker` code at the top of this `README.md` and instantly generate a beautifully styled live website!

</details>
